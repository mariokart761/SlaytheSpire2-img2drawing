## 專案整體技術棧

這個專案主要用的是：

* **Python**：主程式語言。
* **pywebview**：做桌面 GUI，包括按鈕、滑桿、輸入框、狀態欄。
* **OpenCV (`cv2`)**：做影像灰階化、Canny 邊緣偵測、輪廓擷取、模板比對。
* **NumPy**：處理影像陣列、從檔案位元流解碼圖片、截圖轉陣列。
* **Pillow (`PIL`)**：把預覽圖顯示到前端介面上。
* **PyAutoGUI**：讀螢幕尺寸、截圖、移動滑鼠、按下/放開滑鼠、滾輪捲動。
* **keyboard**：註冊全域熱鍵。
* **threading**：把實際繪圖工作丟到背景執行緒，避免 GUI 卡死。
* **PyInstaller**：打包程式為執行檔，且採 GUI 模式。

---

## 各功能與技術細節

### 1. 圖形介面與參數控制

這個功能用 **pywebview**。主視窗可調整大小、可置頂顯示，介面提供：

* 圖片選擇與預覽
* 邊緣偵測閾值（threshold）
* 短線過濾閾值（min length）
* 左右上下邊界避開百分比
* 開始繪製位置
* 拖拽步長
* 起落筆延遲
* 繪圖使用的按鍵選擇
* 暫停恢復時是否自動對齊

**技術判讀：**
這不是遊戲內掛或 API 整合，而是**獨立外掛式桌面工具**，靠 GUI 讓使用者手動校參數後觸發自動畫圖。

---

### 2. 圖片載入與中文路徑相容處理

載入圖片後，程式不是直接用 `cv2.imread()`，而是用：

* `np.fromfile(...)`
* `cv2.imdecode(...)`

這種組合通常是為了**避免 Windows 路徑含中文時 `imread` 失敗**。

支援的格式包含 `jpg / jpeg / png / bmp`等格式。載入成功後會立刻重新解析並更新預覽。

---

### 3. 圖像轉線稿：邊緣偵測

自動繪圖的核心前處理是：

1. 讀入原圖
2. 轉成灰階
3. 用 **Canny edge detection** 抽出輪廓邊緣

**這個功能的技術特性：**

* 閾值越低，抓到的邊緣越多，細節越多，但噪點與繪圖時間也會增加。
* 閾值越高，畫面更簡化，繪圖速度更快。

---

### 4. 輪廓擷取與去噪

邊緣圖做完後，把線段抓出來，再過濾過短輪廓。也就是說，「過濾短線」其實是在做**小輪廓去噪**。

所以這功能不是單純視覺選項，而是直接影響**輪廓數量、畫線穩定性、執行時間**。

---

### 5. 路徑平滑與輪廓簡化

抓到輪廓後，選擇**高保真輪廓逼近**，希望盡量保留曲線和細部。

**這代表的技術取向是：**

* 不追求極端壓縮點數
* 優先保留髮絲、嘴部、小曲線這類細節
* 代價是點數更多、滑鼠運動更頻繁、總耗時更長

也就是說，這個工具不是在做「向量化重建」，而是做**偏保真、偏實用的輪廓近似**。

---

### 6. 路徑排序最佳化：減少空跑

輪廓抓完後，做路徑最佳化。

這是個很關鍵的實作細節。它能減少「抬筆後長距離移動」的時間，也能降低飛線、誤拖、跨區域亂拉的機率。

---

### 7. 預覽功能

預覽不是顯示原圖，而是顯示經 Canny 產生的 `edges`。之後用 Pillow 轉成 `Image`，再縮圖實現預覽。

**用途上有兩個：**

* 讓使用者即時知道 threshold / min_len 調整後實際會畫出什麼
* 避免直接在遊戲裡試錯，先在 GUI 裡校參數

這種「參數即時預覽」對此類工具很重要，因為畫錯一次成本很高。

---

### 8. 繪製區域映射與自適應縮放

真正開始畫時，程式先讀取螢幕解析度，再依使用者設定的上、下、左、右避開百分比，算出一個實際可畫的矩形區域。
把原圖等比縮放到這個矩形內，並居中放置。

**這個功能的本質是螢幕座標映射：**

* 圖片座標 `(img_x, img_y)`
* 經比例換算與偏移後
* 變成螢幕座標 `(screen_x, screen_y)`

好處是：

* 不要求固定解析度
* 不要求固定圖片尺寸
* 使用者只要調整 UI 上的「避開區域」即可適配不同畫面配置

---

### 9. 自動滑鼠繪圖

真正畫線是用 **PyAutoGUI** 模擬滑鼠。流程是：

* 到線段起點前先 `mouseUp`
* 移動到正確位置
* `mouseDown`
* 沿路徑逐點 `moveTo`
* 一段輪廓結束後 `mouseUp`

程式預設使用 `right` (滑鼠右鍵)作為繪圖按鍵，也提供選項更改為滑鼠左鍵。

---

### 10. 拖拽步長控制：用插值分段避免亂線

它不是從上一點直接瞬移到下一點，而是會先算兩點距離，再把路徑切成多個小步：

* 逐步插值出 `nx, ny`
* 每一步都 `pyautogui.moveTo(...)`

這代表「拖拽步長」其實是在控制**滑鼠取樣密度**。

**效果上：**

* 步長小：更平滑、更接近原圖，但更慢
* 步長大：更快，但曲線可能變形，甚至畫面崩壞

---

### 11. 起落筆延遲：用時間節流避免飛線

小心出現「飛線」問題。在落筆、抬筆、移動到起點前後，加入 `time.sleep(draw_delay_time)`；並將下限鎖到 `0.02` 秒 (以 60fps 作為基準)，避免遊戲引擎漏讀輸入事件，該延遲也可供使用者調整，但不得低於0.02s。

**這功能的本質是 I/O 節流：**

* 防止滑鼠事件送太快
* 給遊戲足夠時間消化 mouseUp / mouseDown / moveTo
* 以穩定性換取速度

---

### 12. 熱鍵控制與狀態機

工具支援：

* `F8` 開始
* `F9` 暫停 / 繼續
* `F10` 停止

使用者可以自定義按鍵。做全域熱鍵，內部則用幾個旗標來控制流程。

**這表示它有一個簡單但清楚的執行狀態機：**

* 待機
* 繪圖中
* 暫停
* 強制停止
* 重置 UI

對使用者來說這很重要，因為畫圖通常在遊戲前景進行，不適合一直回工具視窗點按鈕。

---

### 13. 背景執行緒：避免 GUI 卡死

按下開始後，程式不是直接在主執行緒執行繪圖，而是開`Thread`。

這麼做的目的很明確：

* 主執行緒繼續維持 UI 響應
* 狀態欄仍可更新
* 熱鍵和視窗關閉不會完全卡住

這是桌面自動化工具常見而且必要的做法。

---

### 14. 暫停恢復時的自動對齊

當使用者暫停後，程式會：

1. 對繪圖區中央 50% 範圍截一張**Snapshot**
2. 恢復時在當前繪圖區內截圖
3. 用 `cv2.matchTemplate(..., cv2.TM_CCOEFF_NORMED)` 做模板比對
4. 如果找不到，透過上下捲動多輪搜尋
5. 找到後再用滑鼠左鍵實際拖動畫面，把地圖拉回接近原位置
6. 更新繪圖偏移座標(offset)，讓後續筆畫坐標跟著修正 

**這一段的技術組合是：**

* 螢幕截圖
* 影像匹配
* 捲動畫面搜尋
* 物理拖曳校正
* 全域偏移量補償

---

### 15. 偏移補償與繪圖續接

自動對齊成功後，程式會把地圖重新對齊造成的位移記錄到繪圖偏移座標(offset)。

之後每個要畫的點都會加上這組 offset。

這很關鍵，因為如果只把地圖拖回去、但後續路徑還用舊座標，畫面還是會偏掉。

---

### 16. Windows 可執行檔打包

專案使用 **PyInstaller** 打包。

### 17. 前景提取（GrabCut）

載入圖片後，使用者可啟用「前景框選」模式：

1. 預覽圖切換為**彩色原圖**顯示
2. 使用者在預覽圖上**拖曳矩形**框選主體（角色、物件等）
3. 前端將矩形轉換為正規化座標 `[0,1]` 傳給後端
4. 後端以 `cv2.grabCut(..., cv2.GC_INIT_WITH_RECT)` 做前景提取（5 次迭代）
5. 背景像素設為白色，回傳前景影像
6. 後續 Canny 邊緣偵測、輪廓擷取皆使用此前景影像
7. 使用者可按「清除」恢復使用原始圖片

**技術取向：**

* GrabCut 在背景執行緒非同步執行，避免 UI 卡頓
* 框選矩形以正規化座標傳遞，與預覽縮圖尺寸解耦
* 前景提取成功後，透過 `onForegroundDone` CustomEvent 推送新預覽與輪廓數

---

### 18. 指定繪製起點（Anchor 座標）

預設情況下，圖片依避開邊距百分比計算後居中放置於螢幕。啟用「手動指定起點」後：

1. 使用者可直接輸入螢幕 X / Y 座標
2. 或點擊「選取位置」按鈕，倒數 3 秒後自動截取滑鼠所在螢幕座標
3. 此座標作為**縮放後圖片的中心點**（anchor），覆蓋預設的居中計算

**技術實作：**

* `CoordinateMapper` 新增 `anchor: Optional[tuple[int, int]]` 參數
* 指定 anchor 時：`offset_x = anchor_x - scaled_w / 2`、`offset_y = anchor_y - scaled_h / 2`
* 比例尺（scale）仍由避開邊距百分比決定，anchor 只影響位置
* 倒數計時在背景執行緒執行，每秒透過 `onPickCountdown` 推送倒數值，截取後以 `onPickDone` 推送座標
